# Финальный проект: Country Planner

Приложение выросло из клиента REST Countries в offline-first мини-продукт для личного планирования стран. Базовая часть сохранена: список стран, поиск, экран деталей, избранное, навигация, Retrofit, Room, DataStore, StateFlow и Hilt.

## Что добавлено

- Offline-first кэш стран в Room: данные из API сохраняются локально и используются UI как основной источник состояния.
- Фильтры и сортировки главного списка: поиск, избранное, регион, сортировка по названию, населению и площади.
- Нижняя навигация: вкладка `Поиск` с лупой и вкладка `Мои страны` с домиком.
- Экран истории просмотров: открытые страны попадают в Recent, историю можно очистить и из нее можно вернуться в детали.
- Личный план стран: для страны можно сохранить статус и заметку.
- Экран настроек: скрытие посещенных стран, срок актуальности кэша и включение/выключение фоновой синхронизации.
- WorkManager: периодически обновляет локальный кэш стран при наличии сети.

## Новые пользовательские данные

- `country_cache` — локальный кэш стран из API с временем обновления.
- `recent_countries` — история последних открытых стран.
- `country_notes` — личная сущность пользователя: страна, статус (`Хочу посетить`, `В плане`, `Посетил`) и заметка.
- DataStore хранит только настройки приложения: фильтр списка, скрытие посещенных, TTL кэша и настройку фонового обновления.

## Новые сценарии

- Пользователь открывает страну, приложение записывает ее в историю, после чего страна доступна на экране `История`.
- Пользователь добавляет страну в избранное и видит ее в `Мои страны`; если для страны сохранены статус и заметка, они отображаются вместе с карточкой.
- Пользователь настраивает список стран: ищет по названию/столице/региону, фильтрует по региону и сортирует по названию, населению или площади.
- Пользователь включает настройку `Скрывать посещенные`, и страны со статусом `Посетил` исчезают из основного списка.
- Пользователь управляет актуальностью данных: меняет TTL кэша, вручную обновляет кэш или включает фоновую синхронизацию.

## Offline-first

Список стран строится из Room, а сеть используется для обновления кэша. Если кэш уже есть, приложение остается полезным без интернета: список, поиск по локальным данным, история, избранное и личные записи продолжают работать. Если кэша еще нет и сеть недоступна, UI показывает понятную ошибку.

## Фоновая обработка

`CountrySyncWorker` запускается через WorkManager и обновляет `country_cache` в фоне. Работа ставится как unique periodic work с ограничением `NetworkType.CONNECTED`; настройка в DataStore включает или отменяет эту задачу.
